# LLM-Manim V1 UI 设计规范

## 1. 设计方向
V1 UI 采用克制、工具化、线条化的桌面应用风格。界面应服务教师/教研人员的重复工作流：配置、输入、生成、观察状态、查看日志、预览视频。

核心原则：

- 黑白灰为主。
- 彩色只用于状态、风险、强调和关键操作反馈。
- 组件以线条、边框、留白和层级排版组织，不使用拟物或立体效果。
- 信息密度适中，优先清晰、稳定、可扫描。
- 不做营销式、装饰性或卡片堆叠式界面。

## 1.1 设计来源
`references/b_pos8dDmvcka` 是 V1 前端 UI 的主设计参考。实现阶段必须复用该参考中的页面结构、视觉风格、信息层级和核心交互意图。

必须保留的体验特征：

- 黑白灰为主、线条化、紧凑的桌面工具布局。
- 顶部栏展示应用身份、当前项目、Provider/模型和全局入口。
- 左侧项目栏组织项目、新建入口、历史入口和 Provider 设置入口。
- 主工作台聚焦提示词输入、生成状态、视频预览和日志展开。
- 首次启动页聚焦工作区选择和环境检查。
- Provider 设置、基础设置和历史记录作为独立工作视图。
- 状态徽标、日志面板、错误摘要和重试/取消入口保持清晰、靠近任务上下文。

复用规则：

- “完全复用”指复用用户体验、页面结构、视觉层级和交互意图，不表示直接采用引用项目的 Next.js/Tailwind/Radix/shadcn 技术实现。
- 若本文与 `references/b_pos8dDmvcka` 在视觉或交互细节上冲突，优先以引用设计为准。
- 引用设计不得覆盖 V1 产品边界、安全边界和技术栈约束；例如仍不得展示 Manim 源码、API Key 或本地任意绝对路径。

## 2. 色彩
### 2.1 主色
- 背景：白色或接近白色。
- 主文本：接近黑色。
- 次级文本：中性灰。
- 边框/分隔线：浅灰。
- 禁用状态：低对比灰。

### 2.2 语义色
彩色只用于表达语义，不作为大面积装饰：

- 成功：绿色，用于 `succeeded`、连接测试成功、产物可用。
- 运行中：蓝色，用于 `running`、进度提示、当前选中状态。
- 警告：黄色或橙色，用于长任务、明文 Key 风险、依赖缺失预警。
- 错误：红色，用于 `failed`、鉴权失败、渲染失败。
- 取消/中性终止：灰色，用于 `cancelled`。

规则：

- 单个页面不应出现大面积彩色背景。
- 彩色按钮只用于主要动作或危险动作。
- 状态色必须配合文字或图标，不只依赖颜色表达。

## 3. 组件风格
### 3.1 基础组件
- 输入框：白底、细边框、清晰 focus 边框。
- 按钮：线框或实心黑/深灰为主；危险操作使用红色强调。
- 下拉、分段控件、复选框、开关：使用系统化线条组件。
- 表格/列表：用细分隔线和行间距组织，不使用厚重卡片。
- 日志面板：等宽字体、浅灰背景或白底细边框，错误行可使用红色文本/左侧标记。
- 视频预览：保持功能性边框，不做装饰性容器。

### 3.2 禁止项
- 不使用大面积渐变。
- 不使用阴影制造立体卡片。
- 不使用玻璃拟态、毛玻璃、发光、霓虹、3D 装饰。
- 不使用圆角过大的胶囊式组件作为主要视觉语言。
- 不使用装饰性插画、渐变球、漂浮图形。
- 不把多个卡片嵌套成复杂层级。

## 4. 布局
### 4.1 桌面应用布局
推荐基础结构：

```text
左侧导航/项目列表
右侧主工作区
  顶部：项目标题、Provider 选择、主要操作
  中部：提示词输入 / 任务状态 / 视频预览
  底部或侧栏：日志、错误摘要、历史记录
```

规则：

- 页面区域用分隔线和空白区分。
- 固定工具区尺寸，避免任务状态变化造成布局跳动。
- 长文本必须换行或截断，不得溢出容器。
- 任务状态、错误摘要、日志应靠近生成结果，不要分散在多个远离区域。

## 5. 状态表达
任务状态必须清晰可扫描：

| 状态 | 视觉表达 |
| --- | --- |
| `queued` | 灰色标签 |
| `running` | 蓝色标签 + 进度/耗时 |
| `succeeded` | 绿色标签 |
| `failed` | 红色标签 + 错误摘要 |
| `cancelled` | 灰色标签 + 取消说明 |

错误提示必须包含：

- 原因。
- 影响。
- 建议动作。

## 6. 页面要求
### 6.1 首次启动页
- 只聚焦工作区选择和环境状态。
- 明文 Key 风险不在此页提前展示，除非用户进入 Provider 配置。

### 6.2 项目列表页
- 左侧或主区域展示项目列表。
- 每行显示项目名、更新时间、最近任务状态。
- 删除项目使用谨慎的二次确认。

### 6.3 Provider 设置页
- 表单简洁，字段包括 provider_type、base_url、model、api_key。
- API Key 明文存储风险必须用警告色和明确文本提示。
- 连接测试结果使用语义色和短消息。

### 6.4 生成页
- 提示词输入是主区域。
- Provider 选择与生成按钮靠近输入区。
- 任务状态、日志、视频预览形成一条连续工作流。
- 不显示生成的 Manim 源码。

### 6.5 日志与错误页
- 默认展示简化日志。
- 详细日志折叠。
- 错误码可显示，但不能压过用户可读说明。
- 不显示 API Key 或敏感请求头。

## 7. 验收标准
- 页面整体以黑白灰为主，彩色只出现在状态、警告、错误、成功和主要动作中。
- 无明显阴影、立体、玻璃、渐变装饰。
- 表单、列表、日志、按钮均为线条化组件。
- 任务状态不只依赖颜色表达。
- 长提示词、长错误、长日志不破坏布局。
- 视频预览、日志、错误摘要能在同一工作上下文中被用户快速找到。
