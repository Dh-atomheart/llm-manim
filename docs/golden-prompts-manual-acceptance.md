# Golden Prompts 人工验收记录

## 目标

用于 M6 与 V1 发布前的真实人工验收记录。

本文件只记录真实 Provider、真实 API Key、真实 runtime 环境下的执行结果，不记录 mock Provider 或 fake runner 结果。

## 当前状态

- 自动化验收底座已就绪：Rust 单测、Vitest、Playwright 已覆盖核心成功流、Provider 鉴权失败、静态校验失败、running 任务取消与重试。
- 真实人工验收仍需在具备以下条件的环境中执行：
  - 已配置真实 Provider。
  - 本地 `python`、`uv`、`manim`、`ffmpeg`、`ffprobe` 可用。
  - 可实际生成非空 MP4。

## 执行前检查

- 工作区已初始化。
- Provider 已通过“测试连接”。
- 基础设置页 runtime 状态为 `ready`。
- 当前页面不展示 API Key 明文。
- `logs/app.log`、SQLite `job_logs`、错误详情中不存在 API Key 泄露样例。

## 统一通过标准

每个 golden prompt 至少执行 1 次完整流程，并满足：

- `PromptJob.state = succeeded`。
- `render_artifacts` 中存在对应记录。
- MP4 文件存在且非空。
- `duration_secs > 0`。
- 日志中无 `fatal`。
- UI 可直接预览视频。
- “在文件管理器中打开”动作可用。

人工观察项不要求每次都完美，但必须记录：

- 文字是否可读。
- 画面是否明显遮挡或越界。
- 动画节奏是否基本可接受。
- 是否存在明显错误公式、错误几何关系、错误物理方向或错误算法步骤。
